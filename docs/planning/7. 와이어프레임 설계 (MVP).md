---

## 1. 와이어프레임 설계 원칙

1. 모든 화면은 **단일 주요 목적**을 가진다.
2. 면접 중 화면 전환은 최소화한다.
3. 사용자는 현재 단계(설정 / 진행 / 결과 / 이력)를 항상 인지할 수 있어야 한다.
4. AI의 판단 과정은 UI에 노출하지 않는다.

---

## 2. 화면 구조 요약

1. 홈
2. 내정보(JD 설정)
3. 면접 설정
4. 면접 진행
5. 면접 결과
6. 면접 이력 목록
7. 면접 이력 상세

---

## 3. 화면별 와이어프레임 정의

---

## 3-1. 홈 화면

### 화면 목적

- 서비스 진입 및 면접 시작의 단일 진입점 제공

### UI 구성 요소

- 헤더
    - 서비스 로고
- 메인 영역
    - 면접 시작 버튼 (Primary)
    - 안내 문구: “실전처럼 연습하고, 피드백으로 성장하세요.”
- 서브 영역 (선택)
    - 최근 면접 요약 카드

### UX 포인트

- JD 미설정 시 시작 버튼 비활성화
- 비활성화 상태에서 JD 설정 유도 문구 표시

---

## 3-2. 내정보 (JD 설정 화면)

### 화면 목적

- 면접 질문·난이도·평가 기준의 기준점 설정

### UI 구성 요소

- 헤더
    - 화면 타이틀: 내정보 설정
- 입력 영역
    - 직군 선택 (Select)
    - 연차 선택 (Select)
    - 세부 스택 입력 (Tag / Multi Select)
- 하단 영역
    - 저장 버튼 (Primary)

### UX 포인트

- 필수값 미입력 시 저장 버튼 비활성화
- 저장 완료 시 토스트 피드백 제공

---

## 3-3. 면접 설정 화면

### 화면 목적

- 면접 조건을 빠르게 설정하고 시작

### UI 구성 요소

- 설정 섹션
    - 면접 유형 선택 (Radio)
    - 면접 목적 선택 (Checkbox)
    - 면접관 유형 선택 (Radio)
- 정보 섹션
    - 선택 결과 요약 카드
- 하단 영역
    - 면접 시작 버튼 (Primary)

### UX 포인트

- 선택 변경 시 요약 카드 실시간 반영
- 최소 조건 미충족 시 시작 버튼 비활성화

---

## 3-4. 면접 진행 화면

### 화면 목적

- 질문–답변에 몰입할 수 있는 단일 인터랙션 제공

### UI 구성 요소

- 상단 바
    - 질문 진행 상태 (예: 질문 3 / 8)
    - 면접 종료 버튼
- 질문 영역
    - 면접관 질문 말풍선
- 답변 영역
    - 텍스트 입력 필드
    - 음성 답변 시작/종료 버튼
    - 실시간 STT 텍스트 표시 영역
- 하단 영역
    - 답변 제출 버튼

### UX 포인트

- 답변 중 자동 스크롤 방지
- 음성 입력 중 시각적 상태 강조

---

## 3-5. 면접 결과 화면

### 화면 목적

- 점수 없이 학습 중심 피드백 제공

### UI 구성 요소

- 요약 영역
    - 전체 면접 요약 피드백
- 평가 영역
    - 항목별 평가 구간 표시
- 개선 영역
    - 개선 포인트 서술형 카드
- 하단 영역
    - 면접 이력 보기 버튼

### UX 포인트

- 긍정 → 개선 순서로 정보 배치
- 정량 점수 미노출 유지

---

## 3-6. 면접 이력 목록 화면

### 화면 목적

- 과거 면접 이력 탐색

### UI 구성 요소

- 리스트 영역
    - 면접 이력 카드 리스트
        - 날짜
        - 면접 유형
        - 요약 피드백

### UX 포인트

- 최신순 정렬 기본

---

## 3-7. 면접 이력 상세 화면

### 화면 목적

- 특정 면접의 복기 및 학습

### UI 구성 요소

- 상단 요약 영역
    - 면접 개요 정보
- 상세 영역
    - 질문–답변 타임라인
    - 질문별 피드백 카드
- 종합 영역
    - 전체 면접 총평

### UX 포인트

- 타임라인 기반 가독성 확보

---

## 4. MVP 범위 외 UI 명시

- 실시간 코칭 UI 제외
- 타 사용자 비교 UI 제외
- 점수 시각화 UI 제외

---

### 와이어프레임 핵심 문장

> 이 UI는 면접을 평가하지 않는다.
> 
> 
> 사용자가 스스로 돌아볼 수 있게 만든다.
