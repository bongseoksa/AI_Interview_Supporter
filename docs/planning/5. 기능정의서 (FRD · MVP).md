목적은 **개발 착수 가능한 수준으로 화면·기능·데이터·AI 개입 지점을 명확히 정의**하는 것이다.

---

## 1. 기능 정의 원칙

- 모든 기능은 **화면 단위**로 정의한다.
- 사용자 조작 → 시스템 처리 → AI 개입 → 결과 노출 순으로 기술한다.
- 내부 점수는 데이터로만 존재하며 UI에 직접 노출하지 않는다.

## 3. 화면별 기능 정의

---

## F-01. 홈

### 기능 설명

서비스 최초 진입 화면 및 면접 시작 진입점

### 기능 요구사항

- [필수] 면접 시작 버튼 표시
- [필수] JD 미설정 시 버튼 비활성화
- [선택] 최근 면접 요약 표시

### 처리 로직

- JD 존재 여부 확인
- 미존재 시 내정보 화면으로 유도

---

## F-02. 내정보 (JD 설정)

### 기능 설명

면접 질문 생성·난이도·평가 기준 설정

### 입력 항목

- 직군 (필수)
- 연차 (필수)
- 세부 스택 (옵셔널)

### 기능 요구사항

- [필수] JD 저장 기능
- [필수] 필수값 누락 시 저장 불가
- [필수] 수정 시 기존 JD 덮어쓰기

### 데이터 처리

- JD는 면접 시작 시 스냅샷으로 저장

---

## F-03. 면접 설정

### F-03-1. 면접 유형 설정

- 옵션: 1:1 / 다:1 / 다:다
- 기본값: 1:1
- 다:다 선택 시 AI 면접자 자동 포함 (최대 1명)

---

### F-03-2. 면접 목적 설정

- 기술 / 인성 / 상황
- 최소 1개 필수
- 선택 조합에 따라 질문 비율 자동 계산

---

### F-03-3. 면접관 유형 설정

- 기본형 / 압박형 / 유연형
- 기본값: 기본형

---

## F-04. 면접 진행

### 기능 설명

면접 질문–답변 중심 단일 인터랙션 화면

### 기능 요구사항

- [필수] 질문 텍스트 출력
- [필수] 텍스트 답변 입력
- [선택] 음성 답변(STT)
- [필수] 답변 시작/종료 버튼
- [필수] 실시간 음성 텍스트 표시
- [필수] 질문 진행 상태 표시 (n/8)
- [필수] 면접 종료 버튼

### 제약 조건

- 최대 질문 수: 8
- 꼬리 질문: 질문당 최대 2회
- AI 면접자 답변에는 꼬리 질문 미발생

### AI 개입 지점

- 질문 생성
- 꼬리 질문 여부 판단
- 답변 평가(내부 점수)

---

## F-05. 면접 결과

### 기능 설명

정량 점수 비노출 기반 피드백 제공

### 기능 요구사항

- [필수] 면접 요약 피드백
- [필수] 항목별 평가 결과(구간화)
- [필수] 개선 포인트 서술형 제공
- [선택] 이전 면접 대비 변화 코멘트

### 비노출 정책

- 점수(0~100) 직접 노출 금지

---

## F-06. 면접 이력

### F-06-1. 이력 목록

- 표시 항목
    - 면접 일시
    - 면접 유형
    - 면접 목적
    - 요약 피드백

---

### F-06-2. 이력 상세

- 질문/답변 로그
- 질문별 피드백
- 항목별 종합 피드백
- 전체 요약

### 내부 데이터 (비노출)

- 질문별 점수
- 항목별 점수
- 면접관 유형별 평가

---

## 4. 기능 범위 고정 (MVP)

- 실시간 코칭 제외
- 합불 판정 제외
- 사용자 간 비교 제외
