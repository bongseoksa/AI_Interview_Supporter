## 1. 용어 정의

- **면접자**: 서비스를 사용하는 실제 사용자
- **면접관**: AI 면접관

---

## 2. 최소 스펙 (MVP)

### 2-1. JD 설정 (내정보 화면)

JD 정의: JD는 “질문 주제 선정 + 난이도 기준 + 평가 기준”을 결정한다.

- JD는 면접 진행을 위한 **필수 선행 조건**
- JD 구성 요소
    - 직군 (필수)
    - 연차 (필수)
    - 세부 스택 (옵셔널)
- JD가 입력되지 않으면 면접 진행 불가

---

### 2-2. 면접 유형

- 총 3가지
    - 1:1
    - 다:1
    - 다:다
- **기본값: 1:1 면접**
- 다:다 면접에서 다른 면접자는 AI면접자이다.
- AI 면접자는 최대 1명이며, 면접자의 JD와 동일한 수준의 경력을 갖는다.
- 면접자는 AI 면접자에게 답변을 요청하는 버튼 또는 입력을 통해 발화를 유도한다.

---

### 2-3. 면접 목적 및 질문 비율

질문 비율은 **전체 질문 수(최대 8문항)** 기준으로 적용한다.

- 총 3가지
    - 기술면접
    - 인성면접
    - 상황면접
- 기본값: **전체 선택 (기술 50 / 인성 30 / 상황 20)**
- 케이스 기술, 인성 선택: 기술 70 / 인성 30
- 케이스 기술, 상황 선택: 기술 80 / 상황 20
- 케이스 인성, 상황 선택: 인성 50 / 상황 50
- 케이스 (단일 선택): 각 선택 100
- 면접 목적은 선택/해제 가능. 단 최소 1개는 필수
- 목적 내부 우선순위
    1. 기술면접
    2. 인성면접
    3. 상황(케이스)면접

---

## 3. 면접 진행 방식

### 3-1. 인터랙션 방식

- 면접관
    - 텍스트로 질문 및 반응
- 면접자
    - 텍스트 답변 가능
    - 음성 답변 가능

### 3-2. 음성 답변(STT)

- 면접자가 버튼 클릭으로 답변 시작 / 종료
- 음성 입력 중 실시간 텍스트 변환 표시
- STT 기술: **Web Speech API**

---

## 4. 면접관 설정

### 4-1. 면접관 유형

- 기본형: 꼬리물기와 힌트 제공이 균형 잡힌 모델
- 압박형: 꼬리물기 질문 증가
- 유연형: 힌트 제공 증가
- **디폴트: 기본형**
- 면접관은 매 질문에 대한 면접자의 답변을 점수와 근거를 바탕으로 평가하고 저장한다. (이력용으로써 면접 화면에 안보임)
- 답변에 대한 점수범위는 0 ~ 100점이다.
- 각 질문에 대한 점수, 항목별 점수를 가진다.
- 항목은 문제해결, 직무 이해도, 논리 구조로 구성된다.
- 면접관마다의 점수와 판단은 다를 수 있다.
- 면접관은 ai 면접자의 답변에 대한 평가는 하지 않는다.
- 면접관은 ai 면접자가 답변하는 경우, 면접자에게 답변을 요청하며, 이는 질문수에 포함되지 않는다.

### 4-2. 질문 방식

- 반구조화 면접 방식
    - 기본 질문 + 답변 기반 꼬리 질문

---

## 5. 면접 진행 및 종료

- 질문 수는 최대 8문항 까지 진행한다.
- 한 질문에 대한 꼬리물기 질문은 최대 2회 까지 가능하며, 전체 질문 수에 포함되지 않는다.
- 꼬리물기 질문은 ai 면접자의 답변에 대해서는 발생하지 않는다.
- 꼬리물기는 질문에 대한 면접자의 답변이 질문 의도와 부합하거나, 답변 내에서 추가적으로 물을만한 내용이 있는 경우에만 발생한다.
- 면접자는 언제든지 **면접 종료 버튼**으로 종료 가능
- 면접 종료 시
    - 면접 결과 생성
    - 면접 결과 화면 제공

---

## 6. 면접 이력 관리

- 모든 면접 진행 과정은 데이터 로그로 저장
- 질문별, 항목별 면접 점수는 면접관들의 평가 평균으로 한다.
- 총점은 질문별 점수의 평균을 면접 목적 비율로 가중합산하여 계산한다.
- 면접이력 데이터에는 시작시간(YYYY.MM.DD HH:MM), 종료시간(YYYY.MM.DD HH:MM), 면접당시 본인의 JD, 면접 유형, 면접목적, 각 질의응답 정보 및 면접관의 유형별 평가, 질문별 점수 및 피드백, 항목별 점수 및 피드백, 총점 및 면접 전체에 대한 피드백 정보를 가진다.
- 기록 화면에서
    - 지난 면접 이력 목록 확인 가능
    - 각 이력의 요약 정보 확인 가능
- 면접 이력 항목 선택 시
    - 해당 면접의 상세 진행 내역 확인 가능
